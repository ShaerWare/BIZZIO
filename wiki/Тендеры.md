# Тендеры

Единый интерфейс для просмотра RFQ (Запросов цен) и Аукционов. Контроллер: `TenderController`.

## Концепция

Тендеры не являются отдельной сущностью в БД. Это виртуальное объединение двух типов закупочных процедур:
- **RFQ (Запрос цен)** — конкурентный отбор по взвешенным критериям
- **Аукцион** — обратный аукцион на понижение цены

## Маршруты

| Маршрут | Описание |
|---------|----------|
| GET /tenders | Единый каталог всех тендеров (RFQ + Аукционы) |
| GET /tenders/rules | Правила участия в тендерах |
| GET /my-tenders | Мои тендеры (организатор) |
| GET /my-bids-all | Все мои заявки/ставки |
| GET /my-invitations-all | Все мои приглашения |

## Фильтрация

Единый каталог поддерживает фильтры:
- **Тип**: RFQ / Аукцион / Все
- **Статус**: draft, active, trading, closed, cancelled
- **Поиск**: по названию и номеру
- **Период**: фильтр по датам

Черновики скрываются от пользователей, которые не являются организаторами.

## Карточки тендеров

Для отображения используются компоненты:
- `components/rfq-card.blade.php` — карточка RFQ
- `components/auction-card.blade.php` — карточка аукциона

Каждая карточка показывает:
- Тип (Запрос цен / Аукцион)
- Номер и статус (цветной badge)
- Организатор (с логотипом)
- Описание
- Даты (в зависимости от статуса)
- Количество заявок/ставок

## Сравнение RFQ и Аукциона

| Параметр | RFQ | Аукцион |
|----------|-----|---------|
| Метод выбора | Взвешенный скоринг | Минимальная цена |
| Критерии | Цена + Срок + Аванс | Только цена |
| Фаза торгов | Нет | Да (real-time) |
| Анонимизация | Нет | Да (anonymous_code) |
| Статусы | draft → active → closed | draft → active → trading → closed/cancelled |
| Нумерация | К-ГГММДД-0001 | А-ГГММДД-0001 |
| Шаг ставки | Нет | 0.5%–5% от текущей цены |
| Автозакрытие | По end_date | 20 мин без ставок |
